- Correction mutations MUST use `db.$transaction([...])` with `$executeRaw` statements so the corrections insert and the features/holes mutation roll back together. Correction row first, mutation second, per PRD 2b.

- Prisma client type `@prisma/client` exposes `FeatureType` enum string literals (`'green' | 'fairway' | 'tee_box' | 'bunker' | 'water_hazard'`) for type-change endpoints.
